## Security Policy

Overview

WebGuard is a web-based security tool developed as a school project. It is designed to scan web applications for vulnerabilities and analyze uploaded source code for potential security weaknesses. While basic security measures are implemented, the project is intended for educational purposes only.

This project is not actively maintained for long-term development or feature expansion. However, security issues are still taken seriously.



Project Status

WebGuard is a completed academic project. There are currently no plans for continuous development, upgrades, or feature expansion beyond its current state.

That said, we are open to receiving and addressing valid security vulnerability reports, and we will make reasonable efforts to fix confirmed issues without extending or advancing the project’s scope.



Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

How to Report

* Open an issue in this repository (if appropriate), or
* Contact the maintainers directly

Please include:

* A clear description of the vulnerability
* Steps to reproduce the issue
* Expected vs actual behavior
* Any relevant logs or details



Security Considerations

* User inputs (URLs and file uploads) are validated before processing
* Uploaded files are handled safely and are not executed directly
* File types and sizes are restricted to reduce risk
* The system avoids executing untrusted code
* Basic error handling is implemented to maintain stability



Responsible Use

Users must:

* Only scan systems they have permission to test
* Avoid uploading sensitive or confidential files
* Use the tool for ethical and educational purposes only



Limitations

* The system performs basic vulnerability detection and may not identify all security issues
* It is not intended for production or professional security auditing
* Results should not be considered comprehensive security assessments
